## 무엇을 만들었나
`swgo_crawler`는 SWGO에서 회사명·대표자·홈페이지를 먼저 긁고 그걸로 사업자번호를
보완하는 구조였다. 이번엔 **입력을 회사명 + 홈페이지 2칼럼으로 단순화**하고
사업자등록번호 + 부가정보(대표자·주소·업태·종목·규모·이메일·전화)를 일괄조회하는
독립 도구 `bizno_enricher`(Streamlit)를 신규 제작. 친구가 쓰도록 개인계정
(GitHub johnnyahn + Streamlit Community Cloud)으로 배포 예정.

- `lookup.py` — 조회 핵심 로직(외부 의존 없음, CLI도 가능)
- `app.py` — Streamlit UI: 표 붙여넣기 / 엑셀·CSV 업로드 → 진행바 → 결과표(신뢰도 색상) → 엑셀 다운로드

## 핵심 의사결정 · 이유
- **대표자명 없이 식별해야 함**: 기존 사업자번호 382건 중 245건은 SWGO에서 가져온
  대표자명으로 bizno 검색결과를 마스킹 매칭(`장지훈`↔`장지*`)해 동명업체를 걸렀다.
  새 입력엔 대표자명이 없음 → 다른 식별 신호 필요.
- **실증 테스트로 방향 결정**(가장 어려운 245건 중 20건 샘플):
  - 홈페이지에서 대표자명 직접추출 → 재현 2/20 (홈페이지가 대표자를 잘 안 노출, "인사말/이메일" 노이즈) → **기각**
  - **bizno 회사명검색 + "유일 정확일치" 또는 "홈페이지 도메인 일치"로 식별** → 16/20 정확, 3 미발견, 1 오매칭 → **채택**
- **식별 우선순위**(`lookup.py`):
  1) 홈페이지 푸터/하위페이지에서 사업자번호 직접추출 → 출처 `홈페이지`, 신뢰도 `높음`
  2) bizno 회사명검색 → 정규화 상호가 유일 정확일치면 채택(`유일일치`, `중간`)
  3) 동명 후보 여럿이면 각 후보 article 페이지에 입력 도메인 포함 여부로 식별(`도메인일치`, `높음`)
  4) 못 좁히면 최상위 후보를 참고용 반환(`확인필요`, `낮음`)
  5) 번호 확보 시 bizno 직접조회(article id=사업자번호)로 부가정보 100% 취득
- **신뢰도 칼럼을 출력**: `유일일치(중간)`은 동명업체가 실재하면 오매칭 위험
  (실증 1/20, 예: 무지개연구소가 다른 곳을 집음) → 사용자가 `중간/낮음`을 직접
  검증하도록 표에 색상 표기.
- **기존 자산 재사용**: `swgo_crawler/crawl.py`의 사업자번호 정규식(3-2-5),
  하위페이지 추적, 상호정규화, bizno 직접조회, 안내문구(`※`,`(`) 정리 로직을
  SWGO 비의존 형태로 옮김. `Accept-Language: ko-KR`로 영문 자동전환 사이트 대응.

## 영향 / 후속
- 예상 커버리지: 홈페이지 직접(원래 137건 경로) + 검색경로 ~80% 복원 →
  대략 입력의 78~83%(SWGO 없이 2칼럼만인 점 감안하면 충분). 기존 90%보다 낮음.
- 검증: 홈페이지직접(스튜디오랩·피처링) end-to-end OK, 검색경로 샘플 16/20 OK.
- 배포: 개인 GitHub repo → share.streamlit.io, main=`app.py`, `requirements.txt`로 의존성.
- 후속(원하면): 동명 오매칭 더 줄이려면 유일일치에도 주소/도메인 교차확인 강화,
  bizno rate-limit 대비 건당 sleep 조정, 헤드리스로 JS 푸터 보완.
